# Talkzy

Talkzy is a Flutter-based real-time chat application built with Firebase. It includes user authentication, friend requests, one-to-one messaging, notifications, profile management, privacy settings, and theme customization.

## Features

- Email/password authentication with Firebase Auth
- User profile creation and profile management
- Find people and send friend requests
- Accept, reject, and manage friendships
- Real-time one-to-one chat using Cloud Firestore
- Message read/update support
- Online/offline presence updates
- Push and local notification support
- Profile privacy and blocked contact settings
- Light and dark theme support
- Chat theme customization
- Optimized image loading and caching

## Tech Stack

- **Framework:** Flutter
- **Language:** Dart
- **State management/navigation:** GetX
- **Backend:** Firebase
- **Authentication:** Firebase Auth
- **Database:** Cloud Firestore
- **Notifications:** Firebase Cloud Messaging and Flutter Local Notifications
- **Storage/helpers:** Shared Preferences and Flutter Secure Storage
- **Other packages:** Google Fonts, Translator, Cached Network Image, HTTP

## Project Structure


## Getting Started

### Prerequisites

Install the following tools before running the project:

- Flutter SDK `3.6.0` or newer
- Dart SDK included with Flutter
- Android Studio or Visual Studio Code
- Firebase CLI
- Node.js `22` or compatible version for Cloud Functions
- A configured Firebase project

Check your Flutter setup:

```bash
flutter doctor
```

### Installation

Clone the repository:

```bash
git clone https://github.com/varunvenkatesan/Talkzy.git
cd Talkzy
```

Install Flutter dependencies:

```bash
flutter pub get
```

Install Cloud Functions dependencies:

```bash
cd functions
npm install
cd ..
```

### Firebase Setup

This project uses Firebase Auth, Cloud Firestore, Firebase Messaging, and Firebase configuration generated by FlutterFire.

1. Create or open a Firebase project.
2. Enable **Authentication** with the Email/Password provider.
3. Create a **Cloud Firestore** database.
4. Add Android, iOS, web, macOS, or Windows apps as needed.
5. Install and log in to Firebase CLI:

```bash
npm install -g firebase-tools
firebase login
```

6. Configure FlutterFire if you are using your own Firebase project:

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

7. Deploy Firestore rules and indexes:

```bash
firebase deploy --only firestore
```

On Windows, you can also run:

```powershell
.\deploy_rules.ps1
```

## Running the App

Run on the connected device or emulator:

```bash
flutter run
```

Run on a specific platform:

```bash
flutter run -d chrome
flutter run -d android
flutter run -d windows
```

Build a release APK:

```bash
flutter build apk --release
```

Build for web:

```bash
flutter build web
```

## Firestore Collections

The app works with these main Firestore collections:

- `users`
- `friendRequests`
- `friendships`
- `chats`
- `messages`
- `notifications`

Firestore security rules are defined in `firestore.rules`, and indexes are defined in `firestore.indexes.json`.

## Environment Notes

- Do not commit private API keys, service account files, or production secrets.
- If you switch Firebase projects, regenerate `lib/firebase_options.dart`.
- Android Firebase configuration is stored in `android/app/google-services.json`.
- The app automatically creates required Firestore documents when users register and interact with the chat features.
